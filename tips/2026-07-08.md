## git worktreeで並行開発を始める

通常のgitでは、ブランチを切り替えるたびに作業ファイルが切り替わります。でも worktree を使うと、同じリポジトリから複数の「作業ツリー」を作成でき、別々のフォルダで異なるブランチを同時に開けます。例えば機能Aと機能Bを同時に開発するときに便利です。

- kind: lesson
- term: git worktree＝1つのリポジトリから複数の作業フォルダを作成する機能
- try: 「git worktree add ../project-feature feature-branch というコマンドで新しい作業フォルダを作ってください」
- source: https://docs.claude.com/ja/docs/claude-code

## worktreeの基本的な使い方

まずメインの作業フォルダで `git worktree list` を実行すると、現在のworktreeが一覧表示されます。新しいworktreeを作るには `git worktree add <フォルダ名> <ブランチ名>` とします。別のブランチで作業を始めたいときは、作成した新しいフォルダに移動して開発を進めます。

- kind: skill
- term: worktree list＝現在活動中の全ての作業ツリーを確認するコマンド
- try: 「git worktree list で現在のworktree一覧を表示してください」
- source: https://docs.claude.com/ja/docs/claude-code

## worktreeで隔離された環境を作る

Claude Codeではエージェントツール実行時に `isolation: 'worktree'` と設定すると、エージェントがworktree上の隔離された環境で動作します。これにより他のブランチの作業に影響を与えずに、安全に実験的な変更やテストを行えます。

- kind: md
- term: isolation＝他の作業環境から独立した、影響を受けない実行環境
- try: 「エージェント設定でisolation: 'worktree'を使い、隔離された環境で作業してください」
- source: https://docs.claude.com/ja/docs/claude-code

## worktreeを削除する

作業が終わったworktreeはクリーンアップします。`git worktree remove <フォルダ名>` で削除できます。ブランチの削除は別途 `git branch -d <ブランチ名>` で行います。

- kind: new
- term: worktree remove＝不要になった作業ツリーをリポジトリから削除するコマンド
- try: 「git worktree remove ../project-feature で完了したworktreeを削除してください」
- source: https://docs.claude.com/ja/docs/claude-code
