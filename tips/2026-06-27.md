## CLAUDE.mdとは何か
- kind: lesson
- term: CLAUDE.md＝AIエージェントに「このプロジェクトではこうやって仕事をしてほしい」と書く説明書ファイル
- 本文: プロジェクトのルートに`CLAUDE.md`というMarkdownファイルを作ります。ここに「ファイル構成」「コーディング規則」「テスト方法」など、AIが守るべきルールを日本語で書きます。Claude Codeがこのファイルを読んで、あなたの指示に従うようになります。
- try: プロジェクトフォルダに CLAUDE.md というファイルを作成し、「このプロジェクトは React で書かれています。すべてのコンポーネントは関数型にしてください」と書いてみてください
- source: https://docs.claude.com/ja/docs/claude-code

## 実際に書く3つのセクション
- kind: skill
- term: セクション＝CLAUDE.mdの中身を分ける見出し（プロジェクト概要・ルール・ツール）
- 本文: まず「# プロジェクト名」で何かを書きます。次に「## コーディングルール」で「変数は日本語を避ける」などと書きます。最後に「## 使用ツール」で「テストは Jest で書く」と指定します。3つあれば十分で、詳しすぎる必要はありません。
- try: 「## コーディングルール」と「## テスト方法」の2つのセクションだけで CLAUDE.md を作ってみてください
- source: https://docs.claude.com/ja/docs/claude-code

## CLAUDE.mdを Claude Code に読ませる
- kind: new
- term: 連携＝Claude Code が自動的にあなたのCLAUDE.mdを参考にして仕事をする状態
- 本文: CLAUDE.mdをプロジェクトに保存したら、Claude Code に「このプロジェクトを手伝ってください」と頼むだけです。Claude Code が自動でCLAUDE.mdを見つけ、そこに書かれたルールに従います。毎回「この規則で書いて」と言う必要がなくなります。
- try: CLAUDE.md を保存してから Claude Code に「README を作成してください」と依頼し、ルール通りに動くか確認してみてください
- source: https://docs.claude.com/ja/docs/claude-code

## 更新のコツ
- kind: md
- term: 保守＝プロジェクトが変わったときに CLAUDE.md を編集すること
- 本文: 「新しいライブラリを導入した」「テストの方法が変わった」というときは、すぐ CLAUDE.md を更新しましょう。古いルールが残っていると、Claude Code が間違った動き方をします。Git で管理するプロジェクトなら、CLAUDE.md も一緒にコミットして、チーム全体で共有できます。
- source: https://docs.claude.com/ja/docs/claude-code
