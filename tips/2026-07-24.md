## Git Worktreeとは
Git Worktreeは、1つのリポジトリから複数の独立した作業フォルダを作る機能です。通常はブランチを切り替えるたびにファイルが書き変わりますが、Worktreeなら各ブランチが別フォルダに存在するので、同時に複なスな開発ができます。複数の機能開発やバグ修正を並行して進めるときに便利です。

- kind: lesson
- term: Worktree＝同じリポジトリから複数の独立した作業フォルダを作成する仕組み
- try: `git worktree list` でいまのWorktree一覧を確認してください
- source: https://docs.claude.com/ja/docs/claude-code

## 新しいWorktreeを作成する
既存リポジトリで `git worktree add` コマンドを実行します。新しいフォルダが作られ、指定したブランチの内容がそこにコピーされます。メインフォルダに戻らずに別のブランチで作業を続けられるので、コンテキストスイッチの時間が大幅に短縮されます。

```
git worktree add ../feature-folder feature-branch
```

- kind: skill
- term: コンテキストスイッチ＝異なるタスクに注意を切り替えること
- try: あなたのプロジェクトで、新しいWorktreeを1つ作成してください
- source: https://docs.claude.com/ja/docs/claude-code

## Worktreeで並行開発を始める
作成したWorktreeフォルダに移動して、いつもどおりコードを編集・コミット・プッシュします。別のWorktreeではまったく別のブランチで作業を進めるので、コード競合の心配が減り、複数の機能を同時に進められます。

- kind: md
- term: 並行開発＝複数の機能やバグ修正を同時に進める開発スタイル
- try: 2つ目のWorktreeを作成し、別ブランチで新機能の開発を始めてください
- source: https://docs.claude.com/ja/docs/claude-code

## Claude Codeでの自動化
Claude Codeはエージェントツール実行時に `isolation: 'worktree'` を設定することで、隔離された作業ツリー上でタスクを実行できます。複数のAIエージェントが同時に異なるブランチを処理するとき、ファイル競合を防ぎながら効率的に進められます。

- kind: new
- term: Agent isolation＝エージェントの実行環境を独立させて安全に動かす仕組み
- try: Claude Codeに「このプロジェクトの新機能ブランチでテストコードを作成して」と依頼してください
- source: https://docs.claude.com/ja/docs/claude-code
